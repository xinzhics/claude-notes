## 💡 核心对比

| 特性 | Agent | Command | Skill |
|------|-------|---------|-------|
| **本质** | 独立执行者 | 提示词模板 | 可复用指令 |
| **上下文** | 隔离 | 复用主会话 | 两者皆可 |
| **启动方式** | Agent 工具调用 | `/` 手动触发 | Skill 工具/Agent 预加载 |
| **适用场景** | 复杂多步骤 | 工作流自动化 | 技能封装 |

## 🎯 选择指南

### 什么时候用 Command

| 场景 | 推荐 | 原因 |
|------|------|------|
| 简单一次性任务 | ✅ Command | 轻量级 |
| 常用工作流 | ✅ Command | 快捷 |
| 需要复用主上下文 | ✅ Command | 直接 |

### 什么时候用 Agent

| 场景 | 推荐 | 原因 |
|------|------|------|
| 复杂多步骤任务 | ✅ Agent | 隔离环境 |
| 需要并行执行 | ✅ Agent | 不互相干扰 |
| 需要持久化 | ✅ Agent | 支持 memory |

### 什么时候用 Skill

| 场景 | 推荐 | 原因 |
|------|------|------|
| 可复用技能 | ✅ Skill | 一次定义多次使用 |
| 需要被多个 Agent 共享 | ✅ Skill | 预加载模式 |
| 独立执行特定任务 | ✅ Skill | context: fork |

## 📊 性能对比

| 操作 | Agent | Command | Skill |
|------|-------|---------|-------|
| 启动时间 | 慢（需初始化） | 快（即时） | 快（即时） |
| 上下文清理 | 自动隔离 | 共享 | 可选 |
| 资源占用 | 高 | 低 | 低 |

## 💡 组合使用

```
★ Insight ─────────────────────────────────────
最佳实践 = 组合使用
Command 编排流程
Agent 执行任务
Skill 提供能力
─────────────────────────────────────────────────
```

### 典型架构

```yaml
# Command = 入口 + 协调
/weather-orchestrator
    ↓
# Agent = 执行（预加载 Skill）
weather-agent (skills: [weather-fetcher])
    ↓
# Skill = 能力
weather-fetcher
```
